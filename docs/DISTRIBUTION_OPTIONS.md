# Distribution Options

## 評価軸
- 導入容易性
- 更新容易性
- 権限要件
- CI統合性
- ci統合性
- 保守コスト

## 比較表
| 案 | 方式 | 導入容易性 | 更新容易性 | 権限要件 | CI統合性 | 保守コスト |
|---|---|---|---|---|---|---|
| 案A | GitHub Template Repository | 高 | 中 | 中 | 高 | 低 |
| 案B | PyPI パッケージ配布 | 中 | 高 | 低 | 高 | 中 |
| 案C | Docker イメージ配布 | 低 | 中 | 中 | 中 | 高 |

明示:
- 案A: GitHub Template Repository
- 案B: PyPI パッケージ配布
- 案C: Docker イメージ配布

## 採用案
- 採用案: 案A (GitHub Template Repository)
- 採用理由:
  - 初期導入が最短で、GitHub中心運用に合う。
  - CI統合時に追加ランタイム前提が少ない。
  - 運用側の保守コストが最小。

## 不採用理由
- 案B 不採用理由:
  - パッケージ更新運用と各リポジトリ同期の手順が増える。
- 案C 不採用理由:
  - コンテナ基盤前提となり導入障壁と保守コストが上がる。
